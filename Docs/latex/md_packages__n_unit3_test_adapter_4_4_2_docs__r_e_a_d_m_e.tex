Visual Studio 2022 is the recommended version to build and test the adapter.

Use {\ttfamily .\textbackslash{}build -\/t test} at the command line to build and run complete tests.

To create a package use {\ttfamily .\textbackslash{}build -\/t package}

To run and debug tests on .NET Framework, load {\ttfamily Disable\+App\+Domain.\+runsettings}.

From Visual Studio 2019 version 16.\+2 preview 4 it is possible to run tests against a selected target framework in the test project, so one can use this to run .NET Core tests. An alternative approach is to make use of the command line. If you need to frequently debug into .NET Core tests, you can temporarily switch the order of the {\ttfamily \texorpdfstring{$<$}{<}Target\+Frameworks\texorpdfstring{$>$}{>}} in {\ttfamily NUnit.\+Test\+Adapter.\+Tests.\+csproj}.

The {\ttfamily mock-\/assembly} tests are not for direct running.

See \href{https://github.com/nunit/docs/wiki/Packaging-the-V3-Adapter}{\texttt{ https\+://github.\+com/nunit/docs/wiki/\+Packaging-\/the-\/\+V3-\/\+Adapter}} for more details.\hypertarget{md_packages__n_unit3_test_adapter_4_4_2_docs__r_e_a_d_m_e_autotoc_md179}{}\doxysection{Debugger source-\/stepping}\label{md_packages__n_unit3_test_adapter_4_4_2_docs__r_e_a_d_m_e_autotoc_md179}
